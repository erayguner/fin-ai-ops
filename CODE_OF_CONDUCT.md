# Code of Conduct

This project adopts the **Contributor Covenant, version 2.1**, as the
authoritative code of conduct for every space the project maintains
(issues, pull requests, discussions, chat channels, events, and
mailing lists).

- **Canonical text:** https://www.contributor-covenant.org/version/2/1/code_of_conduct/
- **Plain-text version:** https://www.contributor-covenant.org/version/2/1/code_of_conduct/code_of_conduct.txt

The Contributor Covenant defines the standards we hold every
participant to, the enforcement responsibilities of maintainers, the
scope of where the code applies, and the four-tier enforcement
guidelines (correction → warning → temporary ban → permanent ban). The
full text is intentionally not duplicated here: linking to the
canonical version guarantees we stay in step with any future revisions
the Contributor Covenant project publishes.

## How to report a concern

If you experience or witness behaviour that is incompatible with this
code of conduct, please raise it through one of the following channels.
Reports are treated confidentially.

| Channel | When to use |
|---|---|
| GitHub Security Advisory (private) — <https://github.com/erayguner/fin-ai-ops/security/advisories/new> | Anything that involves credential exposure, harassment, or other sensitive material you do not want in a public issue. |
| Email the maintainer at the address listed in [MAINTAINERS](./MAINTAINERS.md) | Any other CoC concern that should reach the project owners directly. |

Do **not** open a public GitHub issue for CoC reports.

## What happens when a report is filed

1. Maintainers acknowledge the report within **48 hours** of receipt.
2. The reporter is offered the option to remain anonymous to the
   reported party where this is practical.
3. Maintainers consult the Contributor Covenant enforcement guidelines
   to choose the appropriate response.
4. The action taken, the rationale, and any opportunity for appeal are
   communicated back to the reporter.
5. Significant decisions are recorded — anonymised where required — in
   [GOVERNANCE.md](./GOVERNANCE.md) so the community has visibility
   into precedent.

## Maintainer commitments

- Treat every report as worth investigating. No-action outcomes still
  receive a written rationale.
- Recuse where a conflict of interest exists; the next named
  maintainer in [MAINTAINERS](./MAINTAINERS.md) takes over.
- Apply the same standard to maintainers as to first-time contributors.
